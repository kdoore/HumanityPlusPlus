This is a **living documentation of process** — an open-source record of one educator-artist-researcher's exploration of AI-assisted interactive code generation as a medium for expressing ideas about human learning, meaning-making, and information literacy.

It is not a finished product. It is a **process artifact** — which is exactly what the framework it documents would predict it should be.

- What gets lost when the analog steps of drawing, making, and teaching are skipped
- How experienced practitioners build on subconscious models that novice learners don't yet have

## The MPCM Framework

**Material → Process → Context → Meaning**

MPCM is a studio arts framework for understanding how meaning emerges in information flows. It draws on the tradition of studio critique — where work is examined not just for its output but for the conditions that produced it.

### The Four Layers

| Layer | Domain | Question |
|-------|--------|----------|
| **M — Material** | Engineering | What is this made of? Tokens, pixels, signals, feeds |
| **P — Process** | Engineering | What generated or amplified it? Algorithms, ranking, routing, automation |
| **C — Context** | Contested | What surrounds it? Body state, incentives, timing, relation, culture |
| **M — Meaning** | Arts / Education / Human | What is forming in me? Emotion, assumption, identity, understanding |

### The Central Argument

Engineers build M (material) and operate P (process). This is immense power — but **power over Process is not the same as power over Meaning.** Meaning is the domain of living minds: arts education, interdisciplinary inquiry, and the cultivation of imagination.

In AI-mediated information ecosystems, Material and Process are often hidden. Users absorb frames, assumptions, and identity postures without consent — because the M and P layers that shaped what reached them are invisible. **Discernment means making those hidden layers visible.**

This is why arts education, academic freedom, and contemplative practice are not soft additions to technical education. They are structural requirements for cognitive depth.

## The Bridging Spiral — Dashboard Dials SR2

The **Bridging Spiral** is a trauma-informed framework for navigating high-dimensional, nonlinear information ecosystems. It uses the metaphor of a **learning control panel** — four instruments a navigator uses to fly through an information universe without losing orientation.

### The Four Instruments

**♠ Somatic Gyroscope** `🜃 Earth`
The body is the first instrument. Before interpreting, sense. Before deciding, settle. In deep learning terms: the *activation function* — the pointwise nonlinearity that gives the cognitive stack its depth. Remove it and the whole architecture collapses to a single linear transformation. This is the structural argument for trauma-informed practice.

**♦ Cognitive Radar** `🜁 Air`
Sweeps the hypothesis field continuously. Holds multiple models simultaneously. Prevents premature convergence. In deep learning terms: the *linear transformation layer* — powerful, but expressive only in composition with the somatic nonlinearity.

**♥ Relational Compass** `🜄 Water`
Orients by care, dignity, and non-harm across scales: self → group → society → ecology → future. In deep learning terms: the *loss function* — defines what counts as error. Values are not soft. They are the source of the learning gradient.

**♣ Dimensional Integration** `🜂 Fire`
Expands understanding across dimensions over time. Metacognitive capacity is developmental — it cannot be shortcut, only cultivated through analog practice, sleep consolidation, and lived iteration. In deep learning terms: *backpropagation* — the error signal that updates weights across the whole stack.

### Aperture — The Master Dial

**A = bandwidth to perceive + integrate + choose right now.**

Three faces:
- **Aₛ** — Somatic bandwidth: can I sense and settle?
- **A꜀** — Cognitive bandwidth: can I hold more than one model?
- **Aᵣ** — Relational bandwidth: can I include other minds without threat?

**Rule: if A is low, do not force convergence.**

### Imagination and Kindness

**✦ Imagination** is not a dial — it is the negative space between dials. It lives in the gaps where meaning wants to form but has not yet crystallised. Protecting imagination means protecting the right to remain uncertain long enough for something genuinely new to emerge. This is the function of arts education that cannot be automated.

**❄ Kindness Resonance** is the field condition. Not niceness: warm truth + firm limits + commitment to repair. The container that makes imagination safe. Without it, the somatic gyroscope cannot settle, the cognitive radar locks onto threat, the relational compass loses true north, and dimensional integration stalls. In deep learning terms: *regularization* — prevents the model from overfitting to threat patterns and losing generalisability.

## The Deep Learning Parallel

Human meaning-making is a deep architecture:

```
INPUT — raw signal (media, event, social stimulus)
   ↓
♠ SOMATIC NONLINEARITY — body gates what passes forward
   ↓
♦ COGNITIVE LINEAR LAYER — conscious reasoning recombines inputs
   ↓
♥ RELATIONAL LOSS FUNCTION — values define what counts as error
   ↓
♣ DIMENSIONAL BACKPROPAGATION — learning updates across time
   ↓
✦ MEANING — emergent, not delivered
```

The critical insight: **without the somatic nonlinearity, stacking cognitive layers adds no depth.** The stack collapses to a single linear transformation — brittle, reactive, single-narrative. This is what trauma does. This is what high-stress information environments induce. This is what systems do when they fire teachers for teaching emotion.

## The Markov Blanket

A **Markov blanket** is the statistical boundary that defines a self — everything inside is "you," everything outside is "world." The blanket does not wall off the inside; it *mediates* what flows between them.

Key parameters:
- **Kindness** regulates permeability — how signals are received at the boundary
- **Imagination** expands what can be received and how it is transformed inside
- **Aperture** determines how much can be integrated at once
- **Mode 0 (Drift)** — the boundary is passive; frames are absorbed without consent
- **Mode 1 (Inquiry)** — the boundary is active; signals are examined before being integrated

A creative studio space — branches, prisms, silk, feathers, mirrors, hanging crystals — *is* an extended Markov blanket. It shapes what signals reach you and how they land.

## A Note on Process and Learning

### What This Exploration Reveals

These interactive models were generated through conversational AI prompting — not through writing code from scratch. This raises genuine questions about learning that the MPCM framework itself helps examine.

**The experienced practitioner** (this researcher) brings prior subconscious models built through years of teaching HTML, Processing, and programming to artists and technologists. The AI-generated code builds on those latent structures. The conversation is productive because there is a deep model to converse with.

**The novice learner** encountering these same tools without that prior experience faces a different situation. Manipulating parameters in an interactive model is not the same as understanding what generates those parameters. Watching signals cross a membrane is not the same as being able to build a membrane. The gap between *using* a model and *understanding* what produces it is precisely the gap between P (process, which AI can route) and M-meaning (which requires the full cognitive stack to form).

### What Gets Skipped

The analog steps that preceded digital work in traditional teaching:
- Drawing model representations by hand before coding them
- Building physical prototypes before simulating them
- Making mistakes that are immediately visible and correctable
- The slow consolidation of understanding through repetition and sleep

These steps activated the somatic nonlinearity — the activation function — before the cognitive linear layer was asked to perform. AI-generated code can skip this sequence entirely. Whether that is a shortcut or a bypass depends entirely on what the learner brings to the encounter.

### The Open Question

Can interactive parameter exploration substitute for the analog making process in building genuine understanding? Or does it produce a different kind of knowing — useful, but shallower? This repository is, in part, an attempt to find out — documented openly so others can examine the process and draw their own conclusions.
